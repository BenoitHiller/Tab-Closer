Tab closer currently provides two exciting features for your enjoyment:

Predefined Matchers

On the right you can define several different patterns. When you click on our button the extensions will close all the tabs it can find that match the patterns you have put in.
Be sure to look through the regex help before trying it out on anything important. Regular expressions have a few simple quirks that can trip up beginners.

Omnibar Tab Closing

If you type 
c *spacebar*
into the omnibar you will be presented with the tab-closer prompt.
The prompt lets you type in a regular expression that will get run over all of your tabs. Anything that matches it, either in the title or in the url, will be closed.
With Regex matchers all you need to do is type in a regular expression that matches any portion of the url or title you want to close.
If you do not know regular expressions I would strongly suggest you take the time to learn. Rather than attempt to teach you now I will provide instead the following:

How to use Regex without knowing what they are

The simplest way to use regular expressions is just to type in characters and it will match anything with exactly those characters. e.g. "www" will match any site with www in its url.
All of the powerful features of regex come from using special characters. The characters: 
( ) [ ] . { } ? \ * + | ^ $
all serve some special purpose. To use them as normal characters you put a \ in front of them. e.g. "www\.google\.com" will match
I have left in the ability to use chrome's built in system for searching through tabs as opposed to the one I hacked together.
To use chrome matchers your patterns have to have the following form: 
PROTOCOL://HOST/PATH
Where PROTOCOL can be any of:
  • *
  • http
  • https
  • file
  • ftp
The * is a wildcard and it will match any of the other options.

HOST is any valid address. You can use a *. at the begining only to match anything string to the left of the period.

PATH can be any string with wildcards anywhere within it.

You can check out Google's page on the topic for examples or a more thorough description.
